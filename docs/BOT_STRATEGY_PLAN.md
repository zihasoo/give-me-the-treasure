# 봇 전략 개선 방안

이 문서는 현재 봇 구조를 기준으로, 봇을 더 똑똑하게 만들기 위한 개선 아이디어와 구현 순서를 정리한 것이다.

## 현재 구조 요약

현재 봇은 `BotStrategy` 인터페이스를 통해 의사결정을 위임한다.

- `decideSplit`: 5장 카드를 두 묶음으로 나누고, 그중 1장을 뒷면으로 숨긴다.
- `decideChoice`: 상대가 나눈 두 묶음 중 하나를 선택한다.
- `decideHelpers`: 준비 단계에서 사용할 도우미 카드를 고른다.
- `planCashIn`: 환금 단계에서 실행할 행동 계획을 만든다.

기본 구현체인 `HeuristicBotStrategy`는 대체로 "즉시 얻을 수 있는 코인 가치"를 기준으로 판단한다.

- 분할: 상대가 공개 가치가 높은 묶음을 가져간다고 가정한다.
- 선택: 공개 카드만 보고 최고 코인 가치가 높은 묶음을 고른다.
- 도우미 선택: 고정 우선순위로 고른다.
- 환금: 현재 가능한 최고 세트를 반복해서 환금한다.

이 방식은 안정적이고 단순하지만, 장기적인 카드 잠재력, 도우미 시너지, 승리 직전 상황, 블러핑을 충분히 반영하지 못한다.

## 1차 개선 목표

가장 효과가 큰 개선은 다음 세 가지다.

1. 환금 계획 최적화
2. 카드 잠재력 점수 도입
3. 상황형 도우미 사용

이 세 가지는 기존 게임 엔진을 크게 건드리지 않고 `BotStrategy` 구현체 안에서 개선할 수 있다.

## 개선안 1: 환금 계획 최적화

현재 방식은 가장 비싼 세트를 하나 찾고, 그 세트를 제거한 뒤 다시 가장 비싼 세트를 찾는 탐욕 전략이다.

문제는 와일드나 핵심 숫자 카드를 먼저 써버리면 전체 환금 합계가 낮아질 수 있다는 점이다.

개선 방향:

- 가능한 모든 유효 세트 후보를 만든다.
- 서로 카드가 겹치지 않는 세트 조합을 탐색한다.
- 총 점수가 가장 높은 환금 계획을 선택한다.

평가 점수 예시:

```text
세트 기본 코인
+ 도우미 보너스
+ 저주 카드 무료 처리 가치
+ 승리 임박 보너스
- 좋은 카드 소모 페널티
+ 남은 카드 잠재력
```

기대 효과:

- 와일드를 더 가치 있게 사용한다.
- 한 번에 여러 세트를 환금할 때 총 코인이 좋아진다.
- 저주 카드 처리와 세트 환금을 함께 판단할 수 있다.

## 개선안 2: 카드 잠재력 점수

카드를 현재 만들 수 있는 세트 기준으로만 평가하면, 다음 라운드에 강해질 수 있는 패를 버리거나 넘겨줄 수 있다.

카드 잠재력 점수를 별도 유틸로 만들면 분할, 선택, 버리기, 도우미 사용에 공통으로 활용할 수 있다.

점수 요소 예시:

- 같은 숫자 2장 이상 보유
- 같은 색 연속 카드 2장 이상 보유
- 와일드 카드 보유
- 저주 카드와 같은 숫자를 처리할 수 있는 가능성
- 5장 같은 색 연속 세트 가능성
- 4장 같은 숫자 1 세트 가능성

사용처:

- `decideSplit`: 상대에게 줄 묶음의 미래 가치를 낮춘다.
- `decideChoice`: 공개 카드 코인 가치가 낮아도 잠재력이 높은 묶음을 선택한다.
- `planCashIn`: 지금 환금할 카드와 남겨둘 카드를 구분한다.
- `chooseDiscard`: 버릴 카드 선택을 개선한다.

## 개선안 3: 상황형 도우미 선택

현재 도우미 선택은 고정 우선순위다. 실제로는 현재 패와 전략에 따라 가치가 달라진다.

도우미별 개선 방향:

- `LUCKY`: 같은 색 연속 5장 가능성이 높을 때 우선한다.
- `LEO`: 같은 숫자 3장 이상 세트를 자주 만들 수 있을 때 우선한다.
- `CUCKOO`: 같은 색 연속 3장 이상이 자주 나올 때 우선한다.
- `ALPHA`: 숫자 1 카드가 이미 있거나 모을 가능성이 있을 때 우선한다.
- `VIPER`: 저주 카드가 많거나 저주 처리 비용이 부담될 때 사용한다.
- `TUSKER`: 좋은 카드가 많아서 보유 제한을 넘길 위험이 있을 때 사용한다.
- `DOUG`: 현재 패가 막혀 있고 비저주 카드 교체 가치가 높을 때 사용한다.
- `JUNK_DEALER`: 버림 더미의 와일드가 현재 세트를 완성할 수 있을 때 사용한다.
- `CROC_BROTHERS`: 첫 번째 복사 가능 대상이 아니라 기대값이 가장 큰 도우미를 복사한다.

기대 효과:

- 도우미가 낭비되는 상황이 줄어든다.
- 패와 도우미의 시너지가 좋아진다.
- 봇이 고정 패턴이 아니라 상황에 맞춰 판단하는 느낌을 준다.

## 개선안 4: 분할 블러핑

현재 분할 전략은 상대가 공개 가치가 높은 묶음을 고른다고 가정한다.

더 강한 전략은 뒷면 카드를 이용해 공개 정보를 왜곡하는 것이다.

전략 예시:

- 핵심 카드를 뒷면으로 숨겨 실제 가치를 감춘다.
- 공개상 좋아 보이는 미끼 묶음을 만든다.
- 내가 받을 가능성이 높은 묶음에 실제 세트 가치를 몰아준다.
- 상대가 선택할 묶음에는 현재 가치는 높지만 미래 잠재력은 낮은 카드를 둔다.

평가 기준 예시:

```text
내가 받을 것으로 예상되는 묶음의 실제 가치
- 상대가 가져갈 것으로 예상되는 묶음의 실제 가치
+ 상대 선택을 유도하는 공개 가치 차이
+ 내 묶음의 미래 잠재력
- 상대 묶음의 미래 잠재력
```

## 개선안 5: 점수 상황 반영

현재 `CashInContext`에는 팀 코인이 포함되어 있다. 이 값을 활용하면 승리 직전 판단을 개선할 수 있다.

전략 예시:

- 우리 팀이 승리 코인에 가까우면 미래 잠재력보다 즉시 환금을 우선한다.
- 이번 환금으로 승리 가능하면 도우미와 와일드를 아끼지 않는다.
- 아직 초반이면 작은 세트 환금을 보류하고 큰 세트를 노린다.
- 보유 제한이 임박하면 잠재력이 낮은 카드부터 버린다.

추가로 상대 팀 코인과 상대 공개 보유 카드까지 컨텍스트에 넣으면 방해 전략도 가능하다.

예시:

- 상대가 승리 직전이면 즉시 점수 획득보다 상대 방해 도우미를 우선한다.
- 상대가 특정 색 연속 세트를 모으는 듯하면 그 색/숫자 카드를 넘기지 않는다.

## 구현 로드맵

### 1단계: 새 전략 클래스 추가

기존 `HeuristicBotStrategy`는 유지하고, 새 구현체를 추가한다.

후보 이름:

- `SmartBotStrategy`
- `AdvancedBotStrategy`
- `ScoredBotStrategy`

기존 전략을 유지하면 기존 봇과 새 봇을 직접 대전시켜 성능을 비교할 수 있다.

### 2단계: 카드 평가 유틸 추가

예상 클래스:

- `CardPotentialScorer`
- `BotCardEvaluator`
- `BotScore`

담당 기능:

- 카드 묶음의 즉시 환금 가치 계산
- 카드 묶음의 미래 잠재력 계산
- 저주 카드 처리 가치 계산
- 와일드 카드 보존 가치 계산
- 버릴 카드 우선순위 계산

### 3단계: 환금 계획 최적화

`planCashIn`을 개선한다.

작업 내용:

- 현재 보유 카드에서 가능한 모든 세트 후보 생성
- 카드가 겹치지 않는 세트 조합 탐색
- 도우미 보너스를 포함해 총 점수 계산
- 가장 좋은 행동 목록 생성
- 보유 제한 초과 시 버릴 카드 선택 개선

### 4단계: 도우미 판단 개선

`decideHelpers`와 `planCashIn`의 도우미 사용 로직을 개선한다.

작업 내용:

- 고정 우선순위 대신 현재 패 기반 점수 사용
- `CROC_BROTHERS` 복사 대상 기대값 평가
- `JUNK_DEALER` 사용 전 와일드 활용 가능성 확인
- `TUSKER`를 무조건 사용하지 않고 보유 제한과 남은 패 가치에 따라 판단

### 5단계: 분할/선택 개선

`decideSplit`과 `decideChoice`에 잠재력 점수를 반영한다.

작업 내용:

- 공개 가치와 실제 가치 분리
- 뒷면 카드가 상대 선택에 주는 영향 평가
- 상대가 선택할 묶음 예측
- 내가 받게 될 묶음의 실제 가치 최대화

### 6단계: 봇 성능 테스트 추가

현재 헤드리스 봇 게임 테스트가 있으므로, 이를 확장해 전략 비교 테스트를 만들 수 있다.

테스트 아이디어:

- `HeuristicBotStrategy` vs `SmartBotStrategy`를 고정 seed로 여러 판 실행
- 새 봇 승률 측정
- 평균 획득 코인 측정
- 게임이 제한 시간 안에 끝나는지 확인
- 도우미 사용 후 무효 행동이 과도하게 발생하지 않는지 확인

## 우선순위 추천

가장 먼저 할 작업:

1. `SmartBotStrategy` 추가
2. 환금 계획 최적화
3. 버리기 카드 선택 개선
4. 도우미 사용 판단 개선
5. 분할 블러핑 추가
6. 상대 상황 반영

1차 구현만으로도 봇 체감 지능은 크게 좋아질 가능성이 높다. 특히 환금 단계에서 와일드와 도우미를 더 잘 쓰면 플레이어가 보기에도 "생각하고 있다"는 느낌이 난다.

## 주의할 점

- 새 전략은 기존 `BotStrategy` 인터페이스를 유지하는 방향이 좋다.
- 게임 엔진 규칙 로직과 봇 평가 로직은 분리한다.
- 봇이 무효 행동을 많이 제출하지 않도록 각 액션 생성 전에 검증한다.
- 성능 비교를 위해 기존 `HeuristicBotStrategy`는 삭제하지 않는다.
- 전략이 너무 완벽해지면 재미가 떨어질 수 있으므로 난이도별 전략도 고려할 수 있다.
